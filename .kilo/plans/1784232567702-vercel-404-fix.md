# Vercel 404 Fix Plan

## Diagnosis

Local build succeeds cleanly:
- `npm run build` passes with all 16 pages generated
- TypeScript, Tailwind, and Next.js 16.2.10 compile without errors
- Only warning: missing `metadataBase` in layout metadata

The Vercel `404: NOT_FOUND` is **not** the app's custom `not-found.tsx` — it is Vercel's platform-level 404. That means the deployment is not producing a runnable Next.js output, or Vercel cannot serve it.

## Most Likely Root Cause

`output: 'standalone'` in `next.config.ts:4`.

- This setting tells Next.js to emit a standalone Node server bundle.
- On Vercel, the platform already injects its own optimized Next.js runtime/serverless functions.
- `standalone` mode can conflict with Vercel's routing and build output expectations, resulting in a missing/empty deployment and the platform 404.

## Fix Plan

### 1. Update `next.config.ts`
- Remove the `output: 'standalone'` line.
- Keep the rest of the config unchanged.

### 2. Fix metadata warning in `src/app/layout.tsx`
- Add `metadataBase` using the production URL so OG images resolve correctly on Vercel.
- Example: `metadataBase: new URL('https://your-vercel-domain.vercel.app')`

### 3. Verify Vercel environment variables
- In Vercel Dashboard → Project → Settings → Environment Variables:
  - Ensure all variables from `.env.example` are present.
  - Ensure they are toggled for **Build** AND **Production** (not just Production).
  - Variables: `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`, `SUPABASE_SERVICE_ROLE_KEY`, `GROQ_API_KEY`, `UPSTASH_REDIS_REST_URL`, `UPSTASH_REDIS_REST_TOKEN`

### 4. Re-deploy
- Push the config change to GitHub.
- Trigger a new Vercel deployment.
- If Vercel is connected to the repo, it should auto-deploy. Otherwise, use Vercel CLI or dashboard.

### 5. Validate
- Check Vercel deployment logs for a successful build.
- Confirm the deployment URL loads the app.

## Rollback / Alternative

If removing `output: 'standalone'` alone does not fix it:
- Add a `vercel.json` with explicit Next.js routing config:
  ```json
  {
    "framework": "nextjs"
  }
  ```
- Re-deploy.
